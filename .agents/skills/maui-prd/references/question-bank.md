# Question Bank For `maui-prd`

Use this file when the project idea is still broad and the student needs help turning it into a realistic PRD.

Ask only 3-5 questions per round. Prefer the most decision-shaping ones.

## 1. Problem and motivation

- Quale problema concreto vuoi risolvere con questa app?
- Perché questo progetto è utile adesso, in questo corso o contesto?
- Il valore principale è velocità, organizzazione, consultazione dati, oppure supporto decisionale?

## 2. Users and context

- Chi userà l'app: studente, docente, cliente, cittadino, team interno?
- L'utente usa l'app occasionalmente o ogni giorno?
- L'app serve in mobilità, in aula, sul lavoro, oppure a casa?

## 3. MVP boundary

- Qual è il flusso minimo che deve funzionare per considerare valido il progetto?
- Cosa vuoi rimandare esplicitamente a una fase successiva?
- Ci sono funzionalità "nice to have" che rischiano di far crescere troppo lo scope?

## 4. Data and integrations

- L'app usa dati remoti, dati inseriti dall'utente, oppure entrambi?
- Serve login oppure può essere anonima?
- Serve persistenza locale? Se sì, per cosa?
- Ci sono API, file, QR code, fotocamera, GPS, o altre dipendenze esterne?

## 5. Constraints

- C'è una scadenza forte o una demo intermedia?
- Android basta oppure serve anche iOS?
- Ci sono regole del corso su librerie, rete, offline, oppure accessibilità?

## 6. Quality and verification

- Come capiremo che il progetto è riuscito?
- Quali errori devono essere gestiti bene fin dall'MVP?
- Quale livello di documentazione e testing vuoi mantenere durante il progetto?

## 7. Typical non-goals

If the student struggles to define non-goals, test these examples:

- autenticazione avanzata;
- sincronizzazione multi-device;
- pannello admin;
- notifiche push;
- analytics estese;
- supporto tablet o desktop dedicato;
- funzionalità AI avanzate non necessarie all'MVP.
