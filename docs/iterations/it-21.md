## Obiettivo
Correggere tre bug post-release che impedivano ai dipendenti di vedere i propri dati (turni, ferie) e un crash di build Android Release.

## Piano

### Bug 1 — ILLink crash Android Release
`AndroidLinkMode` di default taglia reflection-heavy packages (JWT + Firebase) → crash a runtime.
Fix: `<AndroidLinkMode>SdkOnly</AndroidLinkMode>` in `Turnify.Mobile.csproj` solo per Android Release.

### Bug 2 — Dipendenti vuoti in ShiftDetailViewModel
`GetFromJsonAsync<ShiftEmployeeDto[]>("api/employees")` si aspetta array piatto, ma l'API ritorna `{ data: [...] }`. JsonException silenziosa → lista vuota.
Fix: aggiunto `file sealed class ShiftEmployeePagedResult` con `[JsonPropertyName("data")]`, cambiata deserializzazione.

### Bug 3 — Dipendente non vede i propri turni
`ShiftCalendarPage.xaml` aveva le due Grid admin con `IsVisible="{Binding IsEmployeeMode}"` (true per default) → coprendo la employee shift list con "Nessun dato".
Fix: aggiunte proprietà `IsAdminEmployeeMode` e `IsAdminDayMode` in ViewModel, usate nelle Grid admin in XAML.

### Bug 4 — Dipendente non vede le proprie ferie
`VacationRequestsController.GetVacationRequests` passava `GetUserId()` (= `User.Id`) a `GetVacationRequestsByEmployeeAsync` che si aspetta `Employee.Id`. Sempre 0 risultati.
Fix: aggiunto lookup `_employeeRepository.GetByUserIdAsync(userId)` → `employee.Id`, identico a `ShiftsController`.
